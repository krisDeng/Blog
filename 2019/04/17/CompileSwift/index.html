Bus error 10与 Command CompileSwift failed with a nonzero exit code

Posted on 2019-04-17 | In 技术

今天遇到了奇怪的问题，项目编辑的时候在模拟器、手机中都测试通过了，结果在 Archive 的时候报错了。

[Bus error: 10]、 [Command CompileSwift failed with a nonzero exit code] 这是什么鬼？？

为什么 模拟器、手机中联调都没问题，Archive 时候会报错呢，有什么区别吗？？

分析一：

模拟器、手机联调编译的时候，编译器默认是 Debug 环境，而Archive 是release环境，是不是因为这个呢？？

果然，在 切换成 【Release】配置后，模拟器、手机联调也报错了。

分析二：

[Bus error: 10]、 [command CompileSwift failed with a nonzero exit code] 这是什么鬼？？

首先 根据 Xcode 10适配（Command CompileSwiftSources failed with a nonzero exit code) 方法配置了下文件。

但是好像没什么用处，仍然看不到错误信息。

那[Bus error: 10]呢？ 在google了N多资料后，终于看了一篇 How to fix “Bus error 10” after update to Xcode 10.2
根据文章内容，在[Build Settings]搜索 [SWIFT_OPTIMIZATION_LEVEL]，看到了一下内容。

将Release中配置改成了跟 Debug一样，然后再次编译，成功了！！！

那 SWIFT_OPTIMIZATION_LEVEL 的选项的作用是什么呢？？
再次google下，找了一篇文章评测 Swift 3.0 项目编译优化选项对编译速度的影响

未解之谜

1、那就剩下了未解的 为什么会出现[Bus error: 10] 呢？？ 好像是出在 XCode 10以后的情况。
2、Command CompileSwiftSources failed with a nonzero exit code 这个错误是官方已知的问题，也在Xcode10中才出现。
3、在未设置[SWIFT_OPTIMIZATION_LEVEL]选项前，曾经偶尔几次出现 Release 环境中编译成功了，但是 clean 了之后再编译就失败的问题。

莫非这个是 Xcode10 的bug？？
