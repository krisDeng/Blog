SwiftLint使用

Posted on 2017-06-03 | In 管理

1.介绍

SwiftLint是对Swift语言进行代码规范强制检查的工具，针对规范的违反程度显示warning或error。SwiftLint工作在本地，所以需要安装对应的软件。

2.安装

2.1 使用Homebrew

使用HomeBrew安装时，SwiftLint会被安装到系统环境中，多个工程可以共享。

1. `brew install swiftlint`

安装完成后，需要在Xcode的Build Phases中添加Run Script:

1. `if which swiftlint \>/dev/null; then`
2. ` swiftlint`
3. `else`
4. ` echo "warning: SwiftLint not installed, download from https://github.com/realm/SwiftLint"`
5. `fi`

2.2 使用 CocoaPods

通过CocoaPods安装SwiftLint，则SwiftLint只针对当前工程生效。

将如下代码添加到你的 Podfile ：

1. `pod 'SwiftLint'`

执行pod install安装SwiftLint到Pods/文件夹下，可以通过${PODS_ROOT}/SwiftLint/swiftlint调用SwiftLint。

故在Xcode的Build Phases中添加如下Run Script：

1. `"${PODS\_ROOT}/SwiftLint/swiftlint"`

3. 使用SwiftLint

直接编译Xcode工程会通过Script自动调用SwiftLint，初次使用，可能会出现很多warning和error。

3.1 使用SwiftLint自动修复部分warning或error

3.2 配置规则

可以通过本地创建.swiftlint.yml文件来配置具体的规则，以下是基于官方文件修改的配置文件：

  • disabled_rules: 关闭某些默认开启的规则。
  • opt_in_rules: 一些规则是可选的。

  • whitelist_rules: 不可以和 disabled_rules 或者 opt_in_rules 并列。类似一个白名单，只有在这个列表中的规则才是开启的。

    1. disabled\_rules: \# 执行时排除掉的规则（加入表示 不启用该规则）
    2. - colon
    3. - comma
    4. - control\_statement
    6. opt\_in\_rules: \# 一些规则仅仅是可选的（加入表示 启用该规则）
    7. - empty\_count
    9. included: \# 执行 linting 时包含的路径。如果出现这个–path会被忽略。
    10. - Source
    12. excluded: \# 执行 linting 时忽略的路径。 优先级比included更高。
    13. - Carthage
    14. - Pods
    15. - Source/ExcludedFolder
    16. - Source/ExcludedFile.swift
    18. \# 可配置的规则可以通过这个配置文件来自定义
    19. \# 二进制规则可以设置他们的严格程度
    20. force\_cast: warning \# 隐式（表示force\_cast规则 只会报warning，默认报error ）
    21. force\_try:
    22. severity: warning \# 显式
    24. \# 同时有警告和错误等级的规则，可以只设置它的警告等级
    25. line\_length: 128 \# 隐式（超过128 报warning，不报error）
    27. \# 可以通过一个数组同时进行隐式设置
    28. type\_body\_length:
    29. - 300 \# warning
    30. - 400 \# error
    32. \# 或者也可以同时进行显式设置
    33. file\_length:
    34. warning: 500
    35. error: 1200
    37. \# 命名规则可以设置最小长度和最大程度的警告/错误
    38. \# 此外它们也可以设置排除在外的名字
    39. type\_name: \# 修改具体的小规则
    40. min\_length: 4 \# 只是警告
    41. max\_length: \# 警告和错误
    42. warning: 40
    43. error: 50
    44. excluded: iPhone \# 排除某个名字
    46. identifier\_name:
    47. min\_length: \# 只有最小长度
    48. error: 4 \# 只有错误
    49. allowed\_symbols:
    50. - \_ \# 除了字母和数字，还允许下划线
    51. validates\_start\_with\_lowercase: false \# 忽略变量首字母小写
    52. excluded: \# 排除某些名字
    53. - id
    54. - URL
    55. - GlobalAPIKey
    56. reporter: "xcode" \# 报告类型 (xcode, json, csv, checkstyle, junit, html, emoji)

具体的规则名和规则内容见：官方Rules.md

.swiftlint.yml文件可以在文件夹中嵌套使用，子文件下的.swiftlint.yml中的规则会优先生效。

3.3 代码文件内的规则控制

除了使用配置文件，在代码中还可以通过注释来禁用/启用规则。

  • 禁用局部

    1. // swiftlint:disable colon
    2. let noWarning :String = "" // No warning about colons immediately after variable names!
    3. // swiftlint:enable colon
    4. let hasWarning :String = "" // Warning generated about colons immediately after variable names

  • 整个swift文件禁用规则

直接将禁用规则写在首行注释上：

1. `//`
2. `// URLRouter.swift`
3. `// zhilian`
4. `//`
5. `// Created by deerdev on 2017/11/22.`
6. `// Copyright © 2017年 FX. All rights reserved.`
7. `// swiftlint:disable identifier\_name`
