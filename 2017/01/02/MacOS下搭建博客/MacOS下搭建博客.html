想法

最近想搭建一个技术博客，培养自己的写技术博客习惯。

博客架构选型

GitHub Pages + Hexo

Hexo的搭建网上的文档特别多，建议还是看Hexo官网 。

##踩坑之一：

Hexo 官方建议安装 node.js 的 nvm。
然后在nvm 安装时候，遇到了一堆的问题，再根据网上的一堆解决方法，越陷越深…

个人建议用Homebrew 安装 node.

brew install node

在新版本 MacOS系统中，node安装需要权限，在命令之前 + sudo。

sudo brew install node

运行之后再输入本机密码。

如果之前有其他方式安装过 node 的，可以先卸载node，再用 homebrew方式安装。

##踩坑之二：

参照hexo官网 Hexo 安装hexo时,使用命令 npm install hexo-cli -g 却报没有权限,如下:

node-pre-gyp WARN Hit error EACCES: permission denied, mkdir '/usr/local/lib/node_modules/hexo-cli/node_modules/fsevents/lib/binding/Release/node-v67-darwin-x64' 
gyp ERR! clean error 
gyp ERR! stack Error: EACCES: permission denied, rmdir 'build'

解决方法

第一步,赋予目录权限:

sudo chown -R `whoami` /usr/local/lib/node_modules

第二步,安装hexo:
需要注意的点: 在安装hexo时,不要用 sudo 命令.

$ npm install hexo-cli -g

Hexo 生成博客命令

初始化博客：[hexo init blog] 最后的blog 是要创建的名称，可以任取名字。
在初始化 blog 的层级中，输入 [hexo g],就是生成博客内容。

开发过程中node版本升级后，运行时，报错Error: Node Sass

Error: Node Sass does not yet support your current environment: OS X 64-bit with Unsupported runtime (64) For more information on which environments are supported please see:
  • npm uninstall –save node-sass
  • npm cache clean -f
  • npm install –save node-sass

最后 [hexo s] 开启 Hexo Server 服务。

INFO  Start processing
INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

其中 [http://localhost:4000]就是生成后的博客网页地址，可以在浏览器中查看。

部署

cd [userblog]
hexo deploy

ERROR Deployer not found: git

安装 hexo-deployer-git。

npm install hexo-deployer-git --save
